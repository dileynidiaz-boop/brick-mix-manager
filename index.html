📖 How to use Brick Mix Manager · WP3  ▼

Quick Start

  1. §1 Weekly Schedule — Enter your name, select a preset trial from the 32-treatment table, set production date and reps, then click + Add.
  2. §2 Soil & Residue — Enter soil Atterberg limits (IP and USCS auto-calculate). Select agricultural residue type and its moisture.
  3. §3 Mix Calculation — Select country, number of bricks and wet mass. OM auto-sets by country. Click ⚗ Calculate to get all masses and water balance.
  4. §4 Process Conditions — Record mixing, pressing, curing and test dates.
  5. §5 Allocation & Lab Results — Assign bricks per test (Allocation tab), then enter measured results (Lab Results tab) and send to Statistics.
  6. §6 History — Use the search bar to find previous batches. Click Open to restore all sections from a saved record. Export to CSV.
  7. §7 Statistics — Run one-series stats or one-way ANOVA. Data can be imported from §5.

☁️ Subir a Netlify con base de datos compartida — paso a paso

  1. Crear cuenta en Supabase (base de datos gratis):
    • Ve a supabase.com → Start for free → crea cuenta con Google o email.
    • Haz clic en New project, ponle nombre (p.ej. wp3-bricks), elige región, create a password and clic en Create new project. Espera ~1 min.
  2. Crear las tablas:
    • En el menú izquierdo go to SQL Editor → New query.
    • Copia y pega todo este bloque y haz clic en Run ▶:
    create table wp3_batches (id text primary key, data jsonb, updated_at timestamptz default now());
create table wp3_agenda  (id text primary key, data jsonb, updated_at timestamptz default now());
create table wp3_state   (id text primary key, data jsonb, updated_at timestamptz default now());
alter table wp3_batches enable row level security;
alter table wp3_agenda  enable row level security;
alter table wp3_state   enable row level security;
create policy "public rw" on wp3_batches for all using (true) with check (true);
create policy "public rw" on wp3_agenda  for all using (true) with check (true);
create policy "public rw" on wp3_state   for all using (true) with check (true);
    ✓ Deberías ver "Success. No rows returned".
  3. Activar tiempo real:
    • Ve a Database → Replication (menú izquierdo).
    • En la sección Tables, enable toggle de wp3_batches, wp3_agenda y wp3_state.
  4. Copiar tus credenciales:
    • Ve a Project Settings (icono ⚙️ abajo izquierda) → API.
    • Copia Project URL (empieza con https://xxxxx.supabase.co).
    • Copia anon public key (larga cadena que empieza con eyJ…).
  5. Pegar credenciales en este archivo:
    • Abre este archivo HTML con cualquier editor de texto (Bloc de notas, VS Code…).
    • Find the lines const SUPABASE_URL = '' y const SUPABASE_KEY = '' al inicio del bloque <script>.
    • Pega tus valores entre las comillas. Guarda el archivo como index.html.
  6. Subir a Netlify:
    • Ve a netlify.com → crea cuenta gratis → Add new site → Deploy manually.
    • Arrastra y suelta el archivo index.html en la caja gris. ¡Listo!
    • Netlify te da un enlace tipo https://abc123.netlify.app — share it with your team.
    • Todos anyone who opens the link will see and edit the same data in real real.

Autosave

All form inputs auto-save every 2 seconds — to localStorage and to Supabase when online. When you open the page, the latest shared state is loaded automatically.
